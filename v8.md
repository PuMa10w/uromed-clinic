# V8 — Ultra Premium Upgrade Execution Plan

## Mission
- Bring UroMed to an ultra-premium clinical product standard with zero tolerance for iPhone UX regressions.
- Keep interface clean, compact, and fast for real bedside usage.
- Raise trust via content freshness, transparent sources, and predictable behavior across every section.

## Product Principles
- iPhone-first before desktop refinement.
- Clinical speed over decorative complexity.
- One behavior per interaction pattern (tabs, quickbar, filters, search).
- Release only through quality gates.

## Stream A — Information Architecture and Navigation
- Build a single map for all flows:
  - Home -> Urology/Andrology -> Section -> Disease card -> Tabs -> Back/Close.
- Remove duplicate navigation controls in service pages and disease contexts.
- Keep modal and page behavior aligned:
  - Same back logic.
  - Same sticky offsets.
  - Same bottom action language.
- Add route-level checks to ensure key pages are reachable in <= 3 taps from home.

## Stream B — iPhone Layout Contract
- Define shared mobile contract for all major containers:
  - `max-width: 100%`, `min-width: 0`, `overflow-wrap: anywhere`.
  - Touch targets >= 44px.
  - Safe area paddings using `env(safe-area-inset-*)`.
- Normalize sticky stack:
  - Header layer.
  - Disease tabs layer.
  - Bottom quick actions layer.
- Ensure disease tabs always scroll horizontally and auto-snap on iPhone widths.
- Keep quick actions always docked to bottom, never in reading flow.

## Stream C — Premium Visual System
- Formalize token system:
  - Typography scale.
  - Spacing scale.
  - Radius, border, and shadow tiers.
  - Semantic colors for risk/warning/recommendation states.
- Unify card shells:
  - `summary-card`, `clinical-card`, `table-card`.
- Reduce visual noise:
  - Remove debug-like labels and technical crumbs from user-facing UI.
  - Clamp excessive subtitle lines on mobile.

## Stream D — Disease Content Excellence
- Raise each disease to one clinical structure:
  - Overview, Classification, Symptoms, Red Flags, Diagnostics, Differential, Imaging/Labs, Treatment, Follow-up, Prognosis, Cases, Sources.
- For each disease add:
  - "5-minute visit plan".
  - "Escalation triggers".
  - "Guideline freshness" badge with last reviewed date.
- Enforce source pack coverage:
  - EAU/AUA/RU plus contextual references when available.

## Stream E — Diagnostics and Tables
- Convert dense text blocks into readable premium tables where clinically appropriate:
  - Differential diagnosis.
  - Lab norms.
  - Diagnostic algorithm steps.
  - Treatment line comparison.
- On iPhone render tables as stacked card rows without horizontal overflow.

## Stream F — Drug Intelligence 2.0
- Expand drug catalog coverage by high-demand uro/andrology classes.
- Improve search by mapping synonyms:
  - Trade/generic names.
  - Pharmacodynamic effects.
  - Risk tags (QT, nephrotoxicity, infection risk, etc.).
- Add comparison mode for selected drugs:
  - Positive effects.
  - Negative effects.
  - Contraindication highlights.

## Stream G — Calculators and Questionnaires Pro
- Refactor all questionnaires into premium input grids:
  - Clear scoring rails.
  - Instant interpretation panel.
  - Compact mobile layout.
- Add top demanded clinical tools and keep each with:
  - Input guidance.
  - Result interpretation.
  - Clinical next step hints.

## Stream H — Search and Discovery
- Implement command palette style search for mobile:
  - Diseases, drugs, calculators, sections in one entry point.
- Improve debounce states:
  - "Searching..." state.
  - No false empty-state flashes.
- Group and rank results by clinical utility and frequency.

## Stream I — Reliability and Release Discipline
- Keep one-command gate: `npm run verify:premium`.
- Gate includes:
  - Build.
  - Unit tests.
  - iPhone mobile smoke.
  - Content quality audit.
  - Content completeness audit.
- Add fail-fast policy:
  - No deploy if any gate is red.

## Stream J — Deployment and Observability
- Keep Cloudflare preview deployment per wave.
- Promote to stable alias only after gate + visual pass.
- Track usage telemetry:
  - Search usage.
  - Modal open depth.
  - Calculator completion rate.
  - Most used disease sections.

## Execution Waves
- Wave 1 (Stability Core):
  - Lock verification gate, sticky behavior, iPhone overflow compliance.
- Wave 2 (Clinical Readability):
  - Table conversion for diagnostics/differentials/lab norms in priority diseases.
- Wave 3 (Search + Drugs):
  - Unified search boost and pharmacology compare mode.
- Wave 4 (Final Premium Pass):
  - Full visual consistency sweep and release hardening.

## Acceptance Criteria
- No horizontal overflow on target iPhone matrix.
- Sticky tabs and quick actions never interfere with reading.
- No empty critical modules in disease cards.
- Search and calculators respond predictably on mobile.
- All release gates pass before deployment.
