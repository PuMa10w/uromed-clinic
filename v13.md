# UroMed UltraPremium v13

## Summary
v13 adds a clinical-utility layer on top of the stabilized v12 iPhone shell. The focus is not another visual repaint, but faster clinical action: grouped command search, compact disease action headers, drug-risk intelligence, local-only questionnaire persistence, and stricter premium gates.

## Implemented Scope
- Clinical Command Center: command search now groups shortcuts by clinical intent: nosologies, drugs, calculators, algorithms, and red flags.
- Nosologies 2.0: disease overview gets a compact clinical action header generated from existing disease fields: urgent exclusion, confirmation, first step, and referral/follow-up.
- Drugs Pro: drug reference adds risk filters for QT, nephrotoxicity, fertility, partner pregnancy, anticoagulants, CKD, ESBL, and androgens.
- Tools Premium: questionnaire answers persist locally on the device under `uromed-tool-result:*`; no network transport is introduced.
- Visual Audit Pro: mobile smoke now covers grouped command search, disease action header, drug risk rails, and light/dark visual invariants.
- Clinical Governance v13: added `source-registry-v13.json`, `ClinicalLink`, source freshness fields, and new gate sections: `visual_gate`, `drug_gate`, `source_freshness_gate`.

## Safety Notes
- Existing URLs and deep-link format `?tool=...` are preserved.
- No medical recommendations were newly invented in code; the action header derives from existing disease content.
- Drug filters use existing drug names, tags, indications, monitoring, contraindications, and pharmacodynamics text.

## Release Gate
- `npm run verify:premium` remains the single release gate.
- v13 requires green `ui_gate`, `content_gate`, `clinical_gate`, `deploy_gate`, `discoverability_gate`, `visual_gate`, `drug_gate`, and `source_freshness_gate`.
