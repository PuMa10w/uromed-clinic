# V7 — iPhone Navigation & Adaptive Master Plan

## 1) Navigation IA (single source of truth)
- Build one navigation map for all user flows:
  - Home -> Urology/Andrology -> Disease card -> Disease tabs -> Search -> Back/Close.
- Define hard UI rules:
  - Top sticky region = header + section switch only.
  - Mid content = pure reading area.
  - Bottom sticky region = quick actions only.
- Remove duplicate navigation patterns and keep one mobile behavior per context.

## 2) iPhone Header System
- Keep 2-row mobile header:
  - Row 1: brand + search + theme.
  - Row 2: segmented switch Urology | Andrology.
- Move secondary navigation into drawer.
- Enforce safe-area:
  - top/left/right paddings must respect `env(safe-area-inset-*)`.
- Acceptance:
  - no overlap with hero titles;
  - no left drift at 320/375/390/414/428 widths.

## 3) Bottom Navigation & Modal Quick Actions
- Global bottom nav:
  - stable dock position, blur surface, clear active state, 44px+ tap targets.
- Disease modal quickbar:
  - always fixed to bottom on iPhone;
  - no mid-screen jumps while reading;
  - no overlap conflict with footer/system bars.

## 4) Disease Tabs Behavior (critical)
- Horizontal tabs must always scroll on iPhone with momentum.
- Active tab must auto-center into viewport.
- Tabs must stay under disease header, never float at mid-screen.
- Sticky top offset must be deterministic and capped for compact reading.

## 5) Layout Anti-Overflow Pass
- For all cards/rows/tables/chips:
  - `min-width: 0`, `max-width: 100%`, `overflow-wrap: anywhere`.
- Remove root causes of horizontal scrolling (not masking with global hidden overflow).

## 6) Typography & Reading Rhythm
- Reduce line-length for body content in disease cards.
- Normalize line-height/paragraph spacing across tabs.
- Keep heading hierarchy compact and scan-friendly on iPhone.

## 7) Component Unification
- Standardize visual shells:
  - disease cards, quick-access cards, drug cards, calculator cards.
- Secondary metadata badges must be compact and non-blocking.
- Long labels and related-disease chips must wrap gracefully.

## 8) Search UX Mobile
- Full-screen mobile search sheet with focus-first input.
- Replace false empty state during debounce with loading state.
- Group results by entity:
  - diseases, drugs, calculators, sections.

## 9) Data-heavy blocks (tables over raw text)
- Convert long diagnostic/differential/lab text to compact clinical tables.
- On iPhone render as stacked table-cards, not overflowing wide tables.

## 10) Theme Consistency
- Equal visual depth/contrast in dark and light modes.
- Verify state parity (hover/active/focus/disabled) across both themes.

## 11) QA Matrix (must pass)
- iPhone viewports:
  - 320x568, 375x667, 390x844, 414x896, 428x926, landscape 667x375.
- Pages:
  - home, urology, andrology, disease modal tabs, drugs, calculators, footer.
- Checks:
  - no horizontal scroll;
  - sticky regions stable;
  - quickbar always docked to bottom;
  - search and tab navigation reliable.

## 12) Execution Waves
- Wave A (now): navigation frame + sticky behavior + quickbar docking.
- Wave B: reading layout + table migration for key disease tabs.
- Wave C: search refinement + final visual polish + regression pass.
