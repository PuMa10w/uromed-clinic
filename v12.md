# UroMed UltraPremium Masterplan v12 Implementation

## Release Goal

Big release mode: one consolidated iPhone-first upgrade that keeps the existing URL architecture and data model stable while locking the product into a single Clinical Luxury interface contract.

## Implemented Scope

- Added a v12 UI contract layer in `src/styles/ultraPremiumContract.css` for all public sections: home, urology, andrology, pediatric, drugs, tools, calculators, sitemap, favorites, emergency, surgery, metaphylaxis, glossary and humor.
- Added explicit v12 shell tokens for section width, service width, spacing density and minimum control width.
- Standardized mobile first-screen layout, service shells, card radii, touch targets, reading rail, table wrapping and horizontal rail behavior.
- Preserved the existing sticky hierarchy: top navbar, contextual tabs, fixed disease quickbar and bottom shell navigation.
- Kept clinical content and URLs stable; no medical facts were changed in this release.

## Quality Gates Added

- Mobile smoke now includes `/humor` as part of the public route matrix.
- Added a v12 first-screen test across all public sections to catch navbar overlap, clipped controls and horizontal overflow.
- Added a v12 priority disease test for diagnostics, differential, treatment and follow-up sections to keep text, tables, tabs and fixed quickbar stable on iPhone.
- Extended `premium:gate` so v12 cannot pass unless the CSS contract, route matrix and new smoke tests are present.

## Acceptance Standard

- `scrollWidth <= innerWidth + 2` on the iPhone matrix.
- Touch targets remain iPhone-safe.
- Disease tabs remain horizontally usable.
- Quickbar remains fixed at the bottom and outside the reading body.
- Service pages share one premium shell and do not clip controls.
- Spermogram remains discoverable from tools, command search, sitemap and direct deep-link.

## Deploy Rule

Deploy is allowed only after `npm run verify:premium` passes locally and the Cloudflare preview passes the post-deploy mobile smoke.
