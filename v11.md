# UroMed UltraPremium Execution v11

## Release Goal
Довести продукт до единого ultra-premium уровня без поломки URL, данных, поиска, избранного и карточек нозологий. Приоритет релиза: iPhone UX Lock, затем clinical readability, service utility и governance hardening.

## Wave A: iPhone UX Lock
- Единая sticky-иерархия: `navbar -> disease tabs -> fixed quickbar -> bottom shell nav`.
- Горизонтальные rails (`tabs`, `chips`, `filters`, `calculator tabs`, `drug hints`) всегда скроллятся по X и не расширяют body.
- `modal-mobile-quickbar` всегда закреплен снизу viewport, а `mobile-shell-nav` скрывается при открытой карточке нозологии.
- Длинный клинический текст идет через reading rail: контролируемая ширина, переносы, единый ритм абзацев, списков и таблиц.
- Сервисные страницы используют один mobile shell: плотный первый экран, компактные фильтры, без clipped controls.

## Wave B: Clinical Readability Premium
- Priority A/B карточки переводятся в структурный формат: summary, diagnostics, differential, treatment lines, follow-up.
- Таблицы используют responsive fallback на iPhone: rows превращаются в читаемые card rows без горизонтального скролла страницы.
- Верхний модуль нозологии остается компактным: название, МКБ, статусы и источники не мешают чтению.

## Wave C: Service Pages UltraUtility
- `/tools`, `/calculators`, `/drugs`, `/sitemap`, `/favorites`, `/emergency` используют единый визуальный паттерн.
- Спермограмма доступна из `/tools`, `/calculators?tool=sperm-tree`, поиска и sitemap.
- Опросники и калькуляторы сохраняют поток: ввод -> прогресс -> интерпретация -> следующий клинический шаг.

## Wave D: Governance And Release
- `verify:premium` остается единственным входом в релиз.
- Release blockers: horizontal overflow, sticky/quickbar regression, mojibake, пустые priority-блоки, broken search/nav/favorites.
- Перед promotion обязателен Cloudflare preview smoke на iPhone matrix.
